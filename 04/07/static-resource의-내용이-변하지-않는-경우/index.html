Static Resource의 내용이 변하지 않는 경우.

April 7, 2018 in Memo

The content of the Static Resource does not change. ( CSS, IMG, ETC … )

JSP페이지에 내장된 요소들의 CSS를 변경하기 위해

CSS파일을 수정하던중 처음겪는 경우여서 포스팅 해본다.

1. View 요소에 CSS를 변경하려고 /src/main/resources/static/css 의 경로에 위치한 CSS 파일을 수정했다.

2. CSS파일의 수정사항이 반영되지 않았다.

3. CSS파일을 삭제하거나 이름을 바꿔도 이전의 CSS들이 적용됬다.

처음에는 CSS파일의 경로가 잘못됬나하고 생각해봤는데 그럴일은 없었다.

일단 이전의 CSS파일은 인식되고 있는 상태였기 때문에 경로 문제는 아니었다.

헤드부분에는 이런식으로 경로가 적용되있었는데,

<link rel="stylesheet" href="/css/style.css"/> /src/main/resources/static/css 와 “/css/style.css” 가 맞지않고 다른 경로에있는 css파일과 매칭됬나? 했는데

다른 style.css가 있는것도 아니었다.

크롬탭에서 콘솔로 뭐가 문젠가 20~30분은 해매다가 우연히 해결했다.

문제가 된 이유는 수정된 파일을 배포하더라도,

기존의 이용하던 브라우저에 캐쉬가 남아서(Stale Cache) 수정된 파일이 작동하지 않는것이었다.

내 경우에는 크롬에 내장되있는 캐쉬삭제를 통해 해결했는데,

검색해보니 다른 좋은 해결방법이 있어서 이것도 올려본다.

<link rel="stylesheet" href="/css/style.css"/>
문제가 됐었던 태그의 내용인데, 이렇게 하게되면 Stale Cache에 관한 문제가 계속 발생할 가능성이 생긴다.

그때마다 캐쉬를 삭제하는것 보다는 이 문제를 방지하기위해

<link rel="stylesheet" href="/css/style.css?ver=1.0"/>
이런식으로 version에 대한 쿼리스트링을 추가할수도 있다.

이 경우 version에 대한 정보를 인식받아 .css를 읽어오기 때문에 Stale Cache에 관한 문제점이 없어져 좋은 해결책으로 보인다.

나중에 알게 된건데, 크롬에서는 Ctrl+F5키를 누르면 캐시까지 새로고침한다.
아주 유용하게 쓰고있는 기능 :)
